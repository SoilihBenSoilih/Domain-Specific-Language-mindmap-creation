Tâche 1 : Faire une DSL textuelle génératrice de mindmap avec Xtext ou gmf

Ma solution est la définition d'une grammaire xtext complète qui définit toutes les propriétés et fonctionnalités de notre DSL.
Le coeur du travail est là, et se décompose comme suit:

- création du projet Xtext sur Eclipse Epsilon : File > New > Other > Xtext > Xtext project
il ne faut pas oublier de préciser l'extension de notre DSL.
Pour notre cas, l'extension c'est .mmcm qui signifie MindMapCarteMentale.

- 4 projets se créent et on ouvre le premier puis on cherche le fichier .Xtext dans le dossier source(src) et on y définit la grammaire de notre DSL dedans.

- on fait un clique sur : Xtext > « Run As > Generate Xtext Artifacts »

- suite à quoi on choisit le dossier : « *.ui » et puis on fait « Run As > Launch Runtime Eclipse ».

- Dans la nouvelle instance lancée, il suffit de créer un nouveau projet vide : « File > New > Project » dans lequel on crée un fichier « .mmcm » et Xtext nous propose de configurer le projet en tant que projet Xtext et si on valide, on peut ainsi créer une instance avec l'autocomplétion, la coloration...etc


Cette expérimentation sur base sur cet article.
